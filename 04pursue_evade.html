Combined behaviors
  The Pursue and the Evade are two very similar algorithms. If the Flee is inverse of the Seek, then the same is the Evade for the Pursue. The idea behind this methods is quite simple, but with a flaws. The red vehicle is using the Pursue or the Evade method. The red agent checks the current velocity and the position of his target, then predicts when they are going to overlap with each other. In Pursue method the predicted position is passed to the Seek so the red vehicle will catch the target(the blue vehicle). On the other hand in the Evade the predicted position is passed to the Flee so the red agent can evade the target. Because the methods calculate a future position of the target and only that position is considered in next step, which is passing the target to another method, algorithms are not robust.
  The orange vector pointing out of the center of the vehicle is a visualization of the desired velocity and the violet one shows current velocity of the agent. The green line with the circle describes the point of collision between the two vehicles.
  During simulation it is possible to change direction of the target (the blue vehicle) by clicking with the left click mouse button on the canvas.

pursue(target_position, target_velocity) {
  var distance = this.position.distanceBetweenPoints(target_position);
  var predictedTargetPosition = distance / this.maxspeed;
  var target = target_position.add(target_velocity.multiply(predictedTargetPosition))
  return target;
}
evade(target_position, target_velocity) {
  var distance = this.position.distanceBetweenPoints(target_position);
  var predictedTargetPosition = distance / this.maxspeed;
  var target = target_position.add(target_velocity.multiply(predictedTargetPosition))
  return target;
}
